# Contributing

Every negotiation teaches the playbook something. Contributions that make
the next patient's fight easier are the point of this repo.

## What we want

- **Wins and losses, depersonalized.** "Level 5 ER code disputed down to
  Level 3 at a nonprofit in TX; the phrase that moved them was X" is gold.
  File an issue or PR against `knowledge/tactics.md` or
  `knowledge/objections.md`.
- **New objections + comebacks.** What did the billing office say that
  isn't in `objections.md`? What worked against it?
- **State-specific law.** Charity-care mandates, facility-fee limits,
  medical-debt credit-reporting bans, statutes of limitation. Add to
  `knowledge/rights-and-laws.md` with a citation.
- **Template improvements** — wording that got responses, letters for
  situations we don't cover.
- **Verification fixes.** Laws change and tools move. If a URL is dead or a
  deadline/threshold is stale, that's a high-value PR. Include your source.
- **Skill improvements** for other agent harnesses (Cursor, Codex, etc.) —
  keep `AGENTS.md` canonical.

## Hard rules

1. **No personal information. Ever.** No names, account numbers, bill
   images, dollar amounts tied to identifiable people, provider + date
   combinations that could identify you. Depersonalize before you share —
   and remember your `cases/` folder is gitignored for a reason; never
   force-add it.
2. **No invented facts.** Every legal claim needs a citable source. Every
   tactic should be something that actually happened. When something is
   folk wisdom rather than verified, label it.
3. **No legal-advice language.** We say what the law provides and what
   patients can cite; we don't tell people what "will win" or draft around
   the general-information disclaimer.
4. **Keep the tone.** Letters are firm, factual, boring. Docs are direct
   and written for an agent to execute. No rage, no rants — they don't
   negotiate well.

## Mechanics

Standard GitHub flow: fork → branch → PR with a short description of what
changed and why. For substantive claims, include the source in the PR
description. Issues are welcome for anything you don't want to write
yourself — a good issue titled "They said X and I had no comeback" is a
real contribution.
